SQL Syntax

  • Basic Query Syntax

    SELECT = which variables we are selecting

    SELECT DISTINCT = select one row for each value

    FROM = tables we are selecting from

    WHERE = condition we are modifying by

    ORDER BY = ordering (sorting) by certain variables, ascending or descending

    HAVING = like a where clause, but dealing with aggregates

  • JOINS

    Inner Join: Selects elements present in both tables

    Left Join: Selects elements present in "left" table (first listed), if it doesn't exist in right table, those columns show up as blank

    Right Join: Same concept as left join but reversed

    Full Join: Selects all elements in either table

    *Remember we can join multiple tables at once by repeatedly invoking joins, one after another

  • UNION

    A union basically stacks two tables on top of each other - they need to have the same columns of the same variable types

    If we write "union all", that does not sort the values first.
